     Основная цель лабораторных работ - изучить и практически  ос-
воить принципы программной реализации сервисной обработки выходных
результатов математического моделирования составных частей сложных
объектов (СО), полная математическая модель которых  построена  на
мета-уровне.
     Перед началом выполнения любой лабораторной работы, номер ко-
торой определяется графиком, каждый студент  выполняет  требования
пункта 3. "Задание на экспериментальную часть" каждой лабораторной
работы. Студенты, не подготовленные к работе, к занятиям  не допу-
скаются.
     Для получения зачета  по  работе  каждый  студент предъявляет
преподавателю отчет, оформленный в полном соответствии  с требова-
ниями, перечисленными в методических указаниях к работе,  и прохо-
дит собеседование с преподавателем. Студент должен знать цель  ра-
боты, теоретические сведения, уметь анализировать  полученные  ре-
зультаты как в режиме диалога, так и по листингам решения.
     Студент, не сдавший предыдущей работы,  не может быть допущен
к выполнению следующей.

                     ЛАБОРАТОРНАЯ РАБОТА  N 1

     Оценка характеристик выходного сигнала математической модели
                       составной  части СО.

     1. Цель и задачи работы.

     Изучить и практически реализовать методику  получения  оценок
заданных характеристик выходных сигналов  составных  частей  СО  с
применением методов математического моделирования.
     Освоить принципы программной реализации оценки  характеристик
с возможностью  их  подключения в режимы работы программно-методи-
ческого комплекса, построенного на мета-уровне, для математическо-
го моделирования процессов в СО.

     2. Теоретические сведения

     2.1. Основные положения.

     С применением программно-методических комплексов (ПМК), пост-
роенных на мета-уровне,  выходные результаты математического моде-
лирования составных частей СО,  получают, как правило, в виде вре-
менных функций Sвых = F1(t). Для принятия проектных решений требу-
ется  оценка характеристик выходных сигналов,  которая осуществля-
ется программами сервисной обработки.  Программно-методической ба-
зой для выполнения настоящей работы являются результаты,  получен-
ные при выполнении студентами  лабораторной  работы  "Исследование
методики построения математических моделей сложных объектов на ме-
та-уровне".

     2.2. Принципы сервисной обработки расчетных данных.

     Процесс сервисной обработки расчетных данных отделен  от про-
цесса проведения расчетов  и реализуется  при  помощи  специальных
программных средств ведения диалога с пользователем на уровне язы-
ка описания заданий для ПМК.

     Обработка результатов расчета реализуется по принципу:
- выбор сервисной процедуры;
- указание переменных для обработки;
- задание временного интервала обработки сигналов.

     Источником основной информации для сервисной обработки  явля-
ется файл расчетных данных, содержащий значения  выходных сигналов
рассматриваемой составной части СО в  каждой  расчетной  временной
точке.

     2.3. Типы процедур сервисной обработки

     В инженерной  практике  применяются  различные  типы процедур
сервисной обработки расчетных  данных:  определение  средних, дей-
ствующих и экстремальных значений сигналов,  гармонический  анализ
и т.д. В настоящей работе такими процедурами являются:

     - определение среднего значения выходного сигнала

            Si ср =     Sк / n                                (1)


                  ТК - ТН
где       n  = ────────────────  ,       H - шаг расчета модели СО,
                     Н

ТК - конечное время интервала оценки сигнала, ТН - начальное время
интервала оценки сигнала;
Sк - значение выходного сигнала в к-ой расчетной точке.

     - определение действующего значения выходного сигнала:

          Sig =       Sк                            (2)

где n, Sк - определяются также как и для выражения (1)

     - определение интеграла выходного сигнала:

          Si =     Sк * n*H                         (3)

где n, Sк, H   - определяются также как и для выражения (1).

     3. Задание на экспериментальную часть.

     3.1. Провести расчеты для заданной структуры СО, представлен-
ной в виде составных частей,  с применением ПМК,  построенного  на
мета-уровне.
     3.2. Обеспечить  накопление и запоминание результатов расчета
значений сигналов, подлежащих сервисной обработке.

     4. Задание на оценку характеристик выходных сигналов.

     4.1. Разработать программную  реализацию  заданной  процедуры
сервисной обработки.
     4.2. Выполнить  привязку программы сервисной обработки сигна-
лов к ПМК.
     4.3. Выполнить расчеты по  оценке  экспериментальных  данных,
полученных согласно п. 3.2. с регистрацией результатов в табличной
и графической формах и выводом на монитор и принтер.

     5. Контрольные вопросы.

     5.1. Охарактеризовать  функциональное   назначение   заданных
составных частей СО.
     5.2. Сформулировать задачу построения ММ СО на мета-уровне.
     5.3. В чем заключается суть сервисной обработки сигналов ?
     5.4. Методика адаптации процедур сервисной обработки к ПМК.

     6. Порядок оформления отчета.

     6.1. В отчете указывается цель работы,  исходные  данные  для
выполнения экспериментальной части.
     6.2. Распечатка в табличной форме сигналов,  подлежащих  сер-
висной обработке.
     6.3. Листинг процедуры сервисной обработки.
     6.4. Инструкция  пользователя по подключению сервисных проце-
дур к ПМК.
     6.5. Результат  сервисной обработки.
     6.6. Краткие выводы по выполненной работе.

                     ЛАБОРАТОРНАЯ  РАБОТА N 2

        Построение функциональной зависимости "выход-вход"
          математической модели составной части СО и С.

     1. Цель и задачи работы.

     Изучить и практически освоить методику построения  функциона-
льных зависимостей "выход-вход" составных частей СО  с применением
методов математического моделирования.
     Освоить принципы программной реализации построения функциона-
льных зависимостей с возможностью их подключения  в режимы  работы
программно-методического комплекса,  построенного на  мета-уровне,
для математического моделирования процессов в СО.

     2. Теоретические сведения.

     2.1. Основные положения.

     С применением программно-методических комплексов  (ПМК),  по-
строенных на мета-уровне, выходные результаты математического  мо-
делирования составных частей, получают в виде временных функций.

     Для принятия  проектных  решений  на этапах функционального и
схемотехнического проектирования  часто  используются  зависимости
выходного сигнала от сигналов, поступающих на вход составной части
СО.
     Программно-методической базой для выполнения настоящей работы
являются результаты,  полученные при выполнении студентами лабора-
торной работы "Исследование методики построения математических мо-
делей сложных объектов на мета-уровне".

     2.2. Принцип  построения  функциональных  зависимостей   "вы-
ход-вход".

     Процесс построения функциональных зависимостей
реализуется при помощи специальных программных средств ведения ди-
алога с пользователем на уровне описания заданий для ПМК.

     Построение зависимостей реализуется по принципу:

     - выбор составной части СО,  для которой планируется построе-
ние зависимости;
     - указание переменных для обработки с учетом того, что, в об-
щем случае, составная часть СО имеет несколько входов и выходов;
     - задание временного интервала обработки сигналов.

     Источником основной  информации о данных для построения функ-
циональных зависимостей является файл расчетных данных, содержащий
значения выходных и  входных  сигналов  рассматриваемой  составной
части СО, в каждой расчетной временной точке.

     3. Задание на экспериментальную часть.

     3.1. Провести расчеты заданной структуры СО, представленной в
виде  составных  частей,  с  применением ПМК,  построенного на ме-
та-уровне.
     3.2. Обеспечить  накопление и запоминание результатов расчета
значений сигналов,  используемых для построения функциональных за-
висимостей.

     4. Задание на построение функциональных зависимостей.
     4.1. Разработать программную реализацию  построения  функцио-
нальной зависимости.
     4.2. Выполнить привязку программы  построения  функциональной
зависимости к ПМК.

                      ЛАБОРАТОРНАЯ РАБОТА N 3.

       Построение функциональной  зависимости характеристики
         выходного сигнала от внутреннего параметра СО и С


     1. Цель и задачи работы.

     Изучить и  практически реализовать методику построения функцио-
нальных зависимостей характеристик выходного сигнала от  внутреннего
параметра  СО.
     Освоить принципы программной реализации построения функциональ-
ных  зависимостей  характеристик  выходного сигнала W от внутреннего
параметра r СО W = F3(r) с применением ПМК,  построенного  на  мета-
уровне.

     2. Теоретические сведения.

     2.1. Общие положения.

     При выполнении проектных процедур с применением ПМК, входящих в
состав объектно-ориентированных систем  САПР,  требуется  построение
функциональных зависимостей выходного сигнала (средних, действующих,
экстремальных, гармонических,  значений и т.д.) от значений внутрен-
него параметра СО.
     Эти зависимости используются для принятия  проектных решений  с
целью  определения  дальнейшего маршрута проектирования,  выполнения
расчетов по параметрической  оптимизации  и  пр.
     Программно-методической базой  для  выполнения настоящей работы
являются результаты полученные при выполнении студентами  лаборатор-
ных  работ  "Исследование методики построения математических моделей
сложных объектов на мета-уровне".Оценка характеристик выходного сиг-
нала математической модели составной части СО".

     2.2. Принцип построения функциональных зависимостей W = F3(r).

     Процесс построения  параметрических функциональных зависимостей
реализуется при помощи специальных программных средств ведения  диа-
лога с пользователем на уровне описания заданий для ПМК.
     Построение зависимостей W = F3(r) реализуется по принципу:

     - выбор составной части СО,  для которой планируется построения
       зависимости;
     - выбор процедуры для оценки заданной характеристики  выходного
       сигнала;
     - указание переменных и параметров,  требуемых  для  построения
       зависимостей;
     - задание временного интервала обработки сигналов.


     Источником основной информации о данных для построения функцио-
нальных зависимостей являются файлы расчетных значений выходных сиг-
налов и массив значений исследуемых параметров, которые используются
при этом.


     3. Задание на экспериментальную часть.

     3.1. Провести расчеты заданной структуры СО,  представленной  в
виде составных частей с применением ПМК,  построенного на мета-уров-
не.
     Расчеты выполняются  для каждого из заданных значений исследуе-
мого параметра.

     3.2. Обеспечить накопление и  запоминание  результатов  расчета
значений выходных сигналов, используемых для построения функциональ-
ных зависимостей.
     При этом  обеспечивается и организация массивов значений иссле-
дуемых параметров соответственно для каждого из расчетов.

раметра с обеспечением формирования и запоминания их значений.


     4. Задание на построение функциональных зависимостей W = F3(r).

зультатов в  табличной  и  графических формах и выводом на монитор и
принтер.


     5. Контрольные вопросы.

     5.1. Охарактеризовать    функциональное   назначение   заданных
составных частей СО.
     5.2. Представить  в графической форме алгоритм работы процедуры
построения функциональной зависимости W = F3(r).
     5.3. Что понимается под сервисной обработкой данных?  Дать при-
меры.
     5.4. Как  используются функциональные зависимости выходных сиг-
налов от значений входных сигналов и внутренних параметров?

     6. Порядок оформления отчета.

     6.1. В отчете указывается цель работы,  исходные данные для вы-
полнения экспериментальной части.
     6.2. Распечатка в табличной форме  значений  сигналов,  которые
используются при  построении  функциональной зависимости для каждого
значения исследуемого параметра.
     6.3. Распечатка в табличной форме значений характеристик выход-
ного сигнала и заданного параметра.
     6.4. Листинг процедуры построения зависимости W = F3(r).
     6.5. Инструкция пользователя по построению  функциональной  за-
висимости W = F3(r) составной части СО с применением ПМК.
     6.6. Краткие выводы по выполненной работе.
